Đồng bộ và bất đồng bộ

Đồng bộ

  • Request chờ xử lý xong mới trả kết quả.
  • Dễ hiểu, dễ debug.
  • Dễ chậm khi tác vụ nặng.

Bất đồng bộ

  • Trả kết quả nhanh, xử lý nặng ở nền.
  • Tăng hiệu năng và độ chịu tải.
  • Cần quản lý trạng thái và retry.

RabbitMQ trong eCommerce

Những tác vụ nên đưa vào hàng đợi:

  • Gửi email xác nhận đơn.
  • Đồng bộ đơn hàng sang CRM.
  • Xử lý ảnh sản phẩm.
  • Gửi thông báo nội bộ cho CSKH.
Mục tiêu: không để API checkout chờ các việc nền này hoàn tất.

Mô hình Producer - Queue - Worker

Client -> API Checkout
API -> publish message -> RabbitMQ queue
Worker -> consume message -> xử lý nền
Worker -> ack message -> hoàn tất
  • Producer: đẩy message.
  • Queue: giữ message chờ xử lý.
  • Worker: nhận và xử lý.

Code mẫu producer

channel.sendToQueue(
  "order_jobs",
  Buffer.from(JSON.stringify(payload)),
  { persistent: true }
);

Message chứa orderId, action, timestamp để worker xử lý.

Code mẫu worker

channel.consume("order_jobs", async (msg) => {
  const payload = JSON.parse(msg.content.toString());
  await processJob(payload);
  channel.ack(msg);
});

Worker xử lý xong mới ack. Nếu lỗi, có thể nack để retry hoặc đưa vào dead-letter queue.

Bài tập tại lớp

  • Gửi 10 job từ producer.
  • Worker giả lập xử lý chậm 2 giây/job.
  • Quan sát log để thấy API trả nhanh nhưng job xử lý dần.
